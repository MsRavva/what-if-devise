## 1. ВВЕДЕНИЕ

### 1.1 Актуальность

В современном мире искусственный интеллект становится неотъемлемой частью творческих процессов. Веб-приложение "What If Device" представляет собой инновационное решение для создания альтернативных сценариев историй с использованием передовых технологий машинного обучения.

Актуальность проекта обусловлена:
- Растущим интересом к интерактивному контенту
- Потребностью в инструментах для развития творческого мышления
- Необходимостью доступных AI-решений для широкой аудитории
- Развитием технологий генеративного искусственного интеллекта

### 1.2 Цель работы

Основная цель проекта - создание веб-приложения, которое позволяет пользователям генерировать альтернативные сценарии для своих историй с помощью искусственного интеллекта, предоставляя интуитивно понятный интерфейс и возможности для дальнейшего развития сюжета через интерактивный чат.

### 1.3 Задачи

Для достижения поставленной цели были определены следующие задачи:

1. **Техническая реализация:**
   - Разработка современного веб-интерфейса с использованием Next.js 14
   - Интеграция с AI-сервисами для генерации контента
   - Создание системы аутентификации и авторизации
   - Проектирование и реализация базы данных

2. **Функциональные возможности:**
   - Создание формы для ввода исходных историй
   - Реализация генерации альтернативных сценариев
   - Разработка системы сохранения и управления историями
   - Создание интерактивного чат-интерфейса

3. **Пользовательский опыт:**
   - Разработка адаптивного дизайна
   - Создание интуитивно понятного интерфейса
   - Реализация темной и светлой тем
   - Обеспечение быстродействия приложения

### 1.4 Дорожная карта

Проект был реализован в несколько этапов:

**Этап 1: Планирование и проектирование (Неделя 1-2)**
- Анализ требований
- Выбор технологического стека
- Создание архитектуры приложения
- Проектирование базы данных

**Этап 2: Базовая инфраструктура (Неделя 3-4)**
- Настройка Next.js проекта
- Интеграция с Supabase
- Создание системы аутентификации
- Настройка CI/CD

**Этап 3: Основной функционал (Неделя 5-8)**
- Разработка компонентов интерфейса
- Интеграция с AI-сервисами
- Реализация CRUD операций
- Создание API endpoints

**Этап 4: Дополнительные возможности (Неделя 9-10)**
- Разработка чат-интерфейса

## 2. ХОД РАБОТЫ

### 2.1 Изучение различных видов заданий по математике для детей

*Примечание: Данный раздел адаптирован под контекст проекта генерации сценариев*

На начальном этапе был проведен анализ существующих подходов к генерации творческого контента и альтернативных сценариев. Изучены различные методы:

- **Линейное повествование** - традиционный подход к созданию историй
- **Ветвящиеся сюжеты** - интерактивные истории с множественными концовками
- **Генеративные методы** - использование AI для создания контента
- **Коллаборативное творчество** - совместное создание историй

### 2.2 Анализ аналогов приложений

Был проведен детальный анализ существующих решений в области генерации контента:

**Прямые конкуренты:**
- AI Dungeon - текстовые приключения с AI
- Novel AI - генерация художественных текстов
- ChatGPT - универсальный AI-ассистент

**Косвенные конкуренты:**
- Twine - создание интерактивных историй
- Artbreeder - генерация изображений
- Sudowrite - AI-помощник для писателей

**Выявленные преимущества нашего решения:**
- Специализация на альтернативных сценариях
- Русскоязычный интерфейс
- Интеграция чата для развития сюжета
- Бесплатное использование

### 2.3 Выбор технологического стека

После анализа требований и ограничений был выбран следующий технологический стек:

**Frontend:**
- **Next.js 14** - современный React-фреймворк с App Router
- **TypeScript** - типизированный JavaScript для надежности
- **Tailwind CSS** - utility-first CSS фреймворк
- **shadcn/ui** - компоненты пользовательского интерфейса

**Backend:**
- **Next.js API Routes** - серверная логика
- **Supabase** - Backend-as-a-Service решение
- **PostgreSQL** - реляционная база данных

**AI Integration:**
- **Hugging Face API** - доступ к языковым моделям
- **Qwen/Qwen2.5-72B-Instruct** - основная модель для генерации

**Deployment:**
- **Vercel** - платформа для деплоя Next.js приложений
- **GitHub** - система контроля версий

### 2.4 Проектирование архитектуры

Архитектура приложения построена по принципам:

**Клиент-серверная архитектура:**
```
┌─────────────────┐    ┌─────────────────┐    ┌─────────────────┐
│   Frontend      │    │   Backend       │    │   External      │
│   (Next.js)     │◄──►│   (API Routes)  │◄──►│   Services      │
│                 │    │                 │    │                 │
│ - React         │    │ - Authentication│    │ - Hugging Face  │
│ - TypeScript    │    │ - Business Logic│    │ - Supabase      │
│ - Tailwind CSS  │    │ - Data Access   │    │                 │
└─────────────────┘    └─────────────────┘    └─────────────────┘
```

**Структура базы данных:**
```sql
-- Основные таблицы
stories (истории пользователей)
├── id (UUID, PK)
├── user_id (UUID, FK)
├── title (TEXT)
├── content (TEXT)
└── timestamps

scenarios (сгенерированные сценарии)
├── id (UUID, PK)
├── story_id (UUID, FK)
├── user_id (UUID, FK)
├── question (TEXT)
├── ai_response (TEXT)
└── created_at

chat_sessions (сессии чата)
├── id (UUID, PK)
├── user_id (UUID, FK)
├── title (TEXT)
└── timestamps

chat_messages (сообщения чата)
├── id (UUID, PK)
├── session_id (UUID, FK)
├── user_id (UUID, FK)
├── role (user/assistant)
├── content (TEXT)
└── timestamp
```

### 2.7 Создание приложения

Процесс разработки включал следующие этапы:

**Инициализация проекта:**
```bash
npx create-next-app@latest what-if-devise --typescript --tailwind --eslint
cd what-if-devise
npm install @supabase/supabase-js @huggingface/inference
```

**Структура проекта:**
```
what-if-devise/
├── src/
│   ├── app/              # Next.js App Router
│   │   ├── api/          # API endpoints
│   │   ├── auth/         # Аутентификация
│   │   ├── chat/         # Чат-интерфейс
│   │   ├── history/      # История сценариев
│   │   ├── templates/    # Шаблоны историй
│   │   └── what-if/      # Генератор сценариев
│   ├── components/       # React компоненты
│   │   ├── ui/           # Базовые UI компоненты
│   │   └── ...           # Специализированные компоненты
│   ├── lib/              # Утилиты и сервисы
│   ├── styles/           # Глобальные стили
│   └── types/            # TypeScript типы
├── public/               # Статические файлы
└── docs/                 # Документация
```

**Ключевые компоненты:**

1. **AuthProvider** - контекст аутентификации
2. **ThemeProvider** - управление темами
3. **ChatInterface** - интерфейс чата
4. **StoryEditor** - редактор историй
5. **HistoryList** - список сохраненных историй
