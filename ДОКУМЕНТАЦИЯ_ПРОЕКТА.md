## 1. ВВЕДЕНИЕ

### 1.1 Актуальность

В современном мире искусственный интеллект становится неотъемлемой частью творческих процессов. Веб-приложение "What If Device" представляет собой инновационное решение для создания альтернативных сценариев историй с использованием передовых технологий машинного обучения.

Актуальность проекта обусловлена:
- Растущим интересом к интерактивному контенту
- Потребностью в инструментах для развития творческого мышления
- Необходимостью доступных AI-решений для широкой аудитории
- Развитием технологий генеративного искусственного интеллекта

### 1.2 Цель работы

Создание веб-приложения, которое позволяет пользователям генерировать альтернативные сценарии для своих историй с помощью искусственного интеллекта, предоставляя интуитивно понятный интерфейс и возможности для развития сюжета через интерактивный чат.

### 1.3 Задачи

1. **Техническая реализация:**
   - Разработка веб-интерфейса на Next.js 14
   - Интеграция с AI-сервисами Hugging Face
   - Создание системы аутентификации через Supabase
   - Проектирование базы данных PostgreSQL

2. **Функциональные возможности:**
   - Форма для ввода историй и вопросов "что если"
   - Генерация альтернативных сценариев с помощью AI
   - Система сохранения и управления историями
   - Интерактивный чат-интерфейс
   - 64 готовых шаблона историй

3. **Пользовательский опыт:**
   - Адаптивный дизайн для всех устройств
   - Киберпанк-стилистика интерфейса
   - Темная и светлая темы
   - Высокая производительность

## 2. ХОД РАБОТЫ

### 2.1 Анализ аналогов

Был проведён анализ существующих решений:

| Продукт | Преимущества | Недостатки |
|---------|--------------|------------|
| AI Dungeon | Развитая система, сообщество | Платная модель, англоязычность |
| Novel AI | Качественная генерация | Высокая стоимость |
| ChatGPT | Высокое качество, доступность | Не специализирован на сценариях |

**Преимущества нашего решения:**
- Специализация на альтернативных сценариях
- Русскоязычный интерфейс
- Бесплатный доступ
- Интуитивный интерфейс
- 64 готовых шаблона

### 2.2 Выбор технологий

**Frontend:**
- Next.js 14 (App Router)
- TypeScript
- Tailwind CSS
- React 18

**Backend:**
- Next.js API Routes
- Supabase (PostgreSQL)
- Hugging Face API (Qwen2.5-72B-Instruct)

**Деплой:**
- Vercel
- GitHub

### 2.3 Архитектура проекта

```
what-if-devise/
├── src/
│   ├── app/              # Страницы (App Router)
│   │   ├── api/          # API endpoints
│   │   ├── auth/         # Авторизация
│   │   ├── chat/         # Чат с AI
│   │   ├── history/      # История сценариев
│   │   ├── profile/      # Профиль
│   │   ├── templates/    # 64 шаблона
│   │   └── what-if/      # Генератор сценариев
│   ├── components/       # React компоненты
│   │   ├── cyberpunk/    # Киберпанк UI
│   │   └── ui/           # Базовые компоненты
│   ├── lib/              # Утилиты
│   └── types/            # TypeScript типы
└── docs/                 # Документация
```

**Структура базы данных:**
- `stories` - истории пользователей
- `scenarios` - сгенерированные сценарии
- `chat_sessions` - сессии чата
- `chat_messages` - сообщения чата

### 2.4 Дизайн-система

**Стиль:** Киберпанк (Cyberpunk)

**Цветовая палитра:**
- Основной фон: чёрный (#000000)
- Акцент 1: циан (#06b6d4)
- Акцент 2: розовый (#ec4899)
- Акцент 3: зелёный (#10b981)
- Акцент 4: фиолетовый (#8b5cf6)

**Компоненты:**
- `HologramCard` - карточки с голографическим эффектом
- `GlitchText` - стилизованный текст
- `EnergyButton` - кнопки с неоновыми эффектами
- `AIBrainVisualization` - визуализация AI

**Типографика:**
- Шрифт: моноширинный (font-mono)
- Стиль: uppercase, tracking-wider

### 2.5 Разработка приложения

**Реализованные страницы:**

| Страница | Описание |
|----------|----------|
| `/` | Главная с 6 разделами |
| `/what-if` | Генератор сценариев |
| `/chat/[id]` | Интерактивный чат с AI |
| `/templates` | 64 готовых шаблона |
| `/history` | История сценариев |
| `/profile` | Профиль пользователя |
| `/auth` | Авторизация |

**Ключевые функции:**
- Генерация сценариев через Hugging Face API
- Rate limiting (защита от злоупотреблений)
- Error Boundary (обработка ошибок)
- Toast уведомления
- Адаптивный дизайн

### 2.6 Тестирование

- Функциональное тестирование всех страниц
- Проверка работы API
- Кроссбраузерное тестирование
- Адаптивность на разных устройствах
